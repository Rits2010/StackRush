VS Code Web Workbench
Placeholder host (awaiting official web-min build)

This is a placeholder page for self-hosted VS Code Web. You will replace this folder with the official web-min build.

Until then, the bridge will still respond to messages so your app can compile/run. 

Replacement steps:
1. Download the official VS Code Web build (web-min) for a specific version from the microsoft/vscode releases.
2. Replace the entire /public/vscode directory with the contents of the web-min build (preserve this path).
3. Ensure the entry is available as /vscode/index.html.
4. Keep a reference to host-bridge injection (see below) or add a small script tag to load /vscode/host-bridge.js.

Message bridge shim active. Once real workbench is present, it will consume the same messages.
